# 魔法超人官网 Apple 风格升级需求文档

## 1. 项目概述

### 1.1 项目目标
设计一个具有 Apple 式丝滑购买体验的官网，聚焦于引导用户完成套餐选择或服务定制，提供极致简洁、流畅的用户交互体验。

### 1.2 设计理念
- **极简主义**：大量留白，突出核心内容
- **线性流程**：将复杂决策分解为清晰、线性的步骤
- **即时反馈**：所有交互都有明确的视觉反馈
- **状态同步**：配置变更实时反映在所有相关模块
- **沉浸体验**：单列式布局，聚焦用户注意力

---

## 2. 视觉设计规范

### 2.1 布局框架
- **布局方式**：单列式、沉浸式布局
- **起始点**：顶部高清产品图/服务概念图
- **留白原则**：大量留白，使用微妙阴影和细线区分模块
- **响应式设计**：适配移动端、平板和桌面端

### 2.2 配色方案
| 颜色类型 | 色值 | 使用场景 |
|---------|------|---------|
| 主色调 | 深蓝 | 品牌标识、重要按钮、标题 |
| 辅助色 | 浅蓝 | 次级元素、背景点缀 |
| 中性色 | 深灰 | 正文文字、次要信息 |
| 背景色 | 白色 | 页面背景、卡片背景 |

### 2.3 字体规范
- **字体家族**：现代无衬线字体（如 Inter、SF Pro）
- **标题字号**：
  - H1: 48px / 64px / 80px（响应式）
  - H2: 36px / 48px / 60px（响应式）
  - H3: 28px / 32px / 40px（响应式）
- **正文字号**：16px / 18px
- **辅助文字**：14px / 16px

### 2.4 视觉元素
- **图片规格**：高品质产品/服务图，支持动态切换
- **圆角**：使用统一的圆角（8px - 24px）
- **阴影**：微妙的多层阴影效果
- **边框**：细线边框（1px）区分模块

---

## 3. 核心功能模块

### 3.1 首页（Homepage）

#### 3.1.1 Hero Section
**功能描述**：
- 展示顶部高清产品图/服务概念图
- 清晰的主要行动按钮
- Apple 风格的简洁设计

**交互元素**：
- 主按钮："了解方案"
- 次按钮："立即咨询"
- 背景图：当前产品/服务的高清展示

**动效要求**：
- 按钮hover时有颜色、大小或阴影的即时变化
- 背景图可支持微妙的动态效果

#### 3.1.2 价值主张展示
**内容要求**：
- 使用"有力动词 + 核心名词"的 Apple 风格文案
- 示例："精准引流"、"私域沉淀"
- 每个要点配简短的利益点说明

**视觉呈现**：
- 图标 + 文字的组合形式
- 清晰的视觉层级

---

### 3.2 方案/产品配置页（核心页面）

#### 3.2.1 配置流程设计

**线性步骤**：
```
步骤 1 → 步骤 2 → 步骤 3 → ...
```

**示例流程**：
1. 选择目标平台（抖音 / 小红书 / 微信 / 多平台）
2. 选择服务深度（基础运营 / 深度代运营 / 定制化方案）
3. 选择增值工具（AI设计 / 数据分析 / CRM系统）

**交互逻辑**：
- 每个步骤有清晰的视觉分隔
- 完成当前步骤后自动高亮下一步
- 支持返回修改上一步的选择

#### 3.2.2 选项配置模块

**设计要求**：
- 每个选项有清晰的标题和说明
- 选项卡/单选框/复选框样式统一
- 选中状态有明确的视觉反馈
- 支持图文并茂的选项展示

**即时反馈机制**：
- hover：颜色、大小、阴影变化
- click：选中状态高亮，伴随平滑过渡动画

#### 3.2.3 固定摘要面板

**位置**：页面右侧固定悬浮（移动端为底部固定）

**显示内容**：
- 当前选择的套餐/方案概览
- 实时价格更新
- 包含的服务项列表
- 动态参数（如：覆盖平台数、服务时长等）

**交互元素**：
- "下一步" / "购买"按钮（始终可见）
- 价格显示区域（高亮突出）
- 可折叠/展开的详细配置清单

**响应式适配**：
- 桌面端：右侧固定面板
- 移动端：底部固定栏（可展开查看详情）

---

### 3.3 动态内容更新

#### 3.3.1 实时同步机制

**触发条件**：用户更改任何配置选项

**同步内容**：
1. **产品图片**：根据选择动态切换对应产品/服务图
2. **价格计算**：实时更新总价和分项价格
3. **参数显示**：更新服务规格、时长、覆盖范围等
4. **摘要面板**：同步更新配置清单和总览

**性能要求**：
- 无延迟更新（<100ms）
- 平滑过渡动画
- 避免页面刷新

#### 3.3.2 动效细节

**过渡效果**：
- 图片切换：淡入淡出或滑入滑出
- 价格更新：数字滚动动画
- 参数变化：平滑的数值过渡
- 面板展开：弹性动画效果

**动画时长**：
- 快速反馈：100-200ms
- 标准过渡：300-400ms
- 复杂动画：500-600ms

## 4. 交互与动效规范

### 4.1 即时反馈体系

#### 4.1.1 按钮反馈
| 状态 | 视觉变化 | 动画时长 |
|-----|---------|---------|
| 默认 | 基础样式 | - |
| hover | 颜色加深 + 轻微放大 + 阴影增强 | 200ms |
| click | 按压缩小 + 亮度变化 | 100ms |
| disabled | 灰色 + 透明度降低 | - |

#### 4.1.2 选项卡反馈
| 状态 | 视觉变化 | 动画时长 |
|-----|---------|---------|
| 未选中 | 边框 + 浅色背景 | - |
| hover | 边框颜色加深 | 200ms |
| 选中 | 主色背景 + 白色文字 + 阴影 | 300ms |

### 4.2 页面级动效

#### 4.2.1 滚动动效
- 元素进入视口时淡入并上移
- 固定面板保持平滑跟随
- 价格区域保持恒定可见

#### 4.2.2 状态切换动效
- 选项切换：优雅的颜色渐变
- 步骤切换：平滑滑动效果
- 面板展开：弹性动画

## 6. 技术实现要求

### 6.1 前端技术栈
- **框架**：Next.js 16（已使用）
- **UI库**：Tailwind CSS 4（已使用）
- **动画库**：Framer Motion 或 CSS Transitions
- **状态管理**：React Context / Zustand

### 6.2 性能要求
- 首屏加载时间：<2秒
- 交互响应时间：<100ms
- 图片优化：WebP 格式 + 懒加载
- 动画性能：使用 GPU 加速（transform/opacity）

### 6.3 响应式设计
- 断点设置：
  - 移动端：<768px
  - 平板：768px - 1024px
  - 桌面：>1024px

### 6.4 可访问性
- 支持键盘导航
- ARIA 标签完整
- 对比度符合 WCAG AA 标准

## 7. 数据流设计

### 7.1 配置状态管理
```
State {
  step: number,                    // 当前步骤
  selections: {
    platform: string,             // 目标平台
    serviceLevel: string,         // 服务深度
    addons: string[],             // 增值工具
  },
  totalPrice: number,            // 总价
  currentImage: string,           // 当前显示的图片
}
```

### 7.2 实时更新逻辑
```
用户交互 → 更新 State → 触发重新渲染 → 同步更新UI
    ↓
计算新价格 → 更新价格显示
    ↓
切换对应图片 → 更新图片展示
    ↓
更新摘要面板 → 刷新配置清单
```

## 12. 附录

### 12.1 项目背景
本项目基于魔法超人全域运营SOP手册，旨在通过官网提供专业、高级感的服务配置和购买体验，配合线下运营策略，实现全渠道获客。

### 12.2 目标用户
- 需要校服/服装行业数字化运营解决方案的企业
- 寻求私域流量沉淀和精准引流的品牌方
- 希望获得专业代运营服务的客户

### 12.3 核心价值
- 简化复杂决策过程
- 提供透明、清晰的报价
- 打造专业可信的品牌形象
- 提升用户购买转化率
